## Entitlements (entitlements-api-go) Guidelines

You are working on **entitlements-api-go**, a Go service that proxies subscription, compliance, and entitlement checks to backend Red Hat IT services. It serves `/subscriptions`, `/compliance`, and (obsolete) `/seats` endpoints for the Hybrid Cloud Console.

### Project Structure

```
entitlements-api-go/
  main.go                  # Application entry point: logger init, Sentry init, bundle loading, server launch
  ams/                     # AMS client, interface, mock, and query builder
  api/                     # Generated code from oapi-codegen (*.gen.go, gitignored)
  apispec/                 # OpenAPI 3.0 spec and spec-serving handler
  bop/                     # Back Office Proxy client, interface, and mock
  bundle_sync/             # Standalone CLI tool for syncing bundle SKU config to Feature Service
  bundles/                 # Bundle YAML config (bundles.yml gitignored; bundles.example.yml committed)
  config/                  # Viper-based configuration singleton and certificate loading
  controllers/             # HTTP handlers, shared HTTP client, error mapper, oapi-codegen config files
  server/                  # Router setup (chi), Prometheus middleware, and server launcher
  types/                   # Shared domain types (bundles, features, error responses, compliance)
  test_data/               # Static test fixtures (certs, bundle YAML files)
```

### Build and Test

- **Go version**: This repo requires Go 1.26.5. Switch with: `goenv shell 1.26.5` before building or testing. Installed via goenv in `setup.sh`.
- **Setup script**: Run `bash scripts/setup-entitlements-env.sh <repo-path>` for automated environment setup (Go version switch, code generation, build validation).
- **`make generate`** — Regenerates `api/types.gen.go` and `api/server.gen.go`. MUST run after cloning and after any change to `apispec/api.spec.json`.
- **`make build`** — Builds both `entitlements-api-go` and `bundle-sync` binaries.
- **`make test`** — Runs `make generate` then `go test -v ./...`.
- **`make test-all`** — Runs `make generate` then `go test -v --race -p 1 --coverprofile=coverage.txt --covermode=atomic ./...`. This is what CI runs.
- **`make debug-run`** — Runs with `ENT_DEBUG=1` (enables mock AMS/BOP clients — local dev only).

### CI Pipeline

- **Konflux/Tekton** (`.tekton/`): Primary CI gate. Runs `make generate` then `make test-all` with race detector. PR cannot merge if this fails.
- **GitHub Actions** — `json-yaml-validate`: Validates all JSON/YAML files (excluding `err_bundle.yml`).
- **GitHub Actions** — security scan: Grype/Syft container image vulnerability scan. Does not run Go tests.

### Code Conventions

- Never edit or commit `*.gen.go` files — they are gitignored and regenerated by `make generate`.
- `bundles/bundles.yml` is gitignored. Only `bundles/bundles.example.yml` is committed.
- Config uses Viper with `ENT_` prefix for env vars. `GLITCHTIP_DSN` is the exception (read via `os.Getenv`).
- Singleton initialization patterns: `config.GetConfig()`, `logger.InitLogger()`, `controllers.getClient()`.
- Interfaces named after role, not prefixed with `I` (e.g. `AMSInterface`, `Bop`).
- Import aliases: `l` for logger in controllers, `v1` for OCM SDK types, `ocmErrors` for OCM error types.
- Use `fmt.Errorf` with `%w` for error wrapping — `pkg/errors` is not a dependency.
- Use `controllers.getClient()` for HTTP clients — don't create new `http.Client` instances.

### Common Pitfalls

- Forgetting `make generate` after cloning — tests fail if generated files are missing.
- Mutating package-level vars after startup (`bundleInfo`, `featuresQuery`, `paidFeatureSuffix`, HTTP client singleton) causes races.
- `idObj.User` can be `nil` for Service Accounts — always nil-check before accessing `User.*` fields.
- The seats API is obsolete (`DisableSeatManager: true`) — don't enable it.

### Testing Patterns

- Tests use Ginkgo/Gomega framework.
- Mock patterns: interface-based mocks and function variable mocks coexist.
- `MakeRequest` test helper uses functional options to override identity fields.
- `toPtr[T]` generic helper converts values to pointers for generated API types.

### Two Binaries

1. **`entitlements-api-go`** (`main.go`): The HTTP API server.
2. **`bundle-sync`** (`bundle_sync/main.go`): CLI init container that syncs bundle SKU config to Feature Service. Uses stdlib `log`/`fmt`, not the structured logger. Supports `--dry-run`.
